Story-Telling

Story-Telling, fe.

  • NPC Interaction
  • story events
is done in visiual novel like style similiar to renpy.

Example scenario:

  • (A) You meet a wise man that could give you some information. But a bunch of grunts stole his googles that he needs to decypher his scrolls. This starts a quest QuestWiseMansGoogles
  • (B) After finding the grunt's hideout you have the option to fight them, steal the googles back or persuade them to hand them over.
  • (C) You bring the googles back to the wise man and earn his gratitude.
  • (D) Depending on other occurences, additional dialog themes might get unlocked,

 

  • use a generic asset/scene for dialog
  • trigger the asset and point it to getThemesForDialog(ThemeID). ThemeID is an Identifier for the concrete dialogScene f.e. SceneWiseMan
  • quest/storys/Chatter.. register themself to the DialogMgr:  RegisterDialogForQuest(ThemeID, QuestID)
  • getThemesForDialog fetches the actual quests/character relation/... related to ThemeID and triggers getDialog(ThemeID) to build a combined DialogTree
  • depending on pc choice/calculation: navigate through the dailogtree and switch state of the actual dialog-theme
  • the dialog might be terminated and switch back to the previous scene (exploration,...) or call a new scene (combat, trading,...).

Visual novel functions

The visual novel asset implements

  • dialog display
  • menu selection, custom buttons
  • Edit-fields for Text & Number
  • transition, fading
  • background image
  • multiple layerd images
  • animation of x-y-z, scale, rotation, shake
  • graphic effects (whirl, jitter)

Data

Where is the story data stored?

DialogTreeBuilder

A database that checks witch DialogTrees are applicable for the ThemeID. It therefore iterates over all registered DialogEntrys and returns an initial DialogTree.

 

DialogTree

An object that represents the actual state of the dialog

getDialog() -> DialogSet: returns an DialogSet containing information for the text & pictures to display, animations to play, settings for fade in/out, choices

continueDialog(choiceID) -> result: triggers the creation of the next DialogSet; result might be "continue dialog", "exit dialog", "exit dialog and switch scene"
